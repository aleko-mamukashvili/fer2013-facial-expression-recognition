# Challenges in Representation Learning: Facial Expression Recognition (FER2013)

ეს პროექტი ეძღვნება Kaggle-ის პოპულარულ კონკურსს — Challenges in Representation Learning: FER2013. პროექტის ფარგლებში რეალიზებულია 4 სხვადასხვა სიღრმისეული სწავლების (Deep Learning) არქიტექტურა, რომელთა ევოლუციამაც ეტაპობრივად მოგვცა მარტივი ბაზისური (Baseline) მოდელიდან მოწინავე რეზიდუალური ქსელების (ResNet) დონემდე მისვლის შესაძლებლობა.

* კონკურსის ბმული: [Kaggle Competition](https://www.kaggle.com/competitions/challenges-in-representation-learning-facial-expression-recognition-challenge)
* ექსპერიმენტების თრექინგი: [Weights & Biases Workspace](https://wandb.ai/amamu23-free-university-of-tbilisi-/fer2013-experiments/workspace?nw=nwuseramamu23)

## მონაცემთა ბაზის ანალიზი (Dataset Analysis)

სავარჯიშო სიმრავლე შეიცავს 28,709 გამოსახულებას. მონაცემთა მთავარი პრობლემა არის კრიტიკული დისბალანსი კლასებს შორის:
 Happy (ყველაზე წარმოდგენილი): 7,215 ნიმუში. მოდელები ამ კლასს ყველაზე მარტივად და მაღალი სიზუსტით სწავლობენ.
 Disgust (ყველაზე მცირე): მხოლოდ 436 ნიმუში. მონაცემთა სიმცირის გამო, ყველა საბაზისო არქიტექტურას უჭირს ამ ემოციის სწორად იდენტიფიცირება.

 დასკვნა: კლასების ასეთი არათანაბარი გადანაწილება აიძულებს მოდელს მიკერძოვდეს უმრავლესობის კლასისკენ, რაც მოითხოვს სპეციალური რეგულარიზაციისა და მონაცემთა აუგმენტაციის ტექნიკების გამოყენებას.

## არქიტექტურების დეტალური განხილვა

### 1. SmallCNN — ბაზისური მოდელი (Baseline)
 არქიტექტურა: 2 კონვოლუციური ფენა (8 და 16 ფილტრით), MaxPool, ყოველგვარი რეგულარიზაციის (Dropout, BatchNorm) გარეშე. სრულად დაკავშირებული ფენა (FC): 2304 -> 64 -> 7.
 Forward & Backward ვერიფიკაცია: მოდელის გამომავალი განზომილება ემთხვევა კლასების რაოდენობას [batch, 7]. Loss ფუნქცია სტაბილურად შემცირდა 1.72-დან 1.13-მდე 15 ეპოქაში.

#### ჰიპერპარამეტრების ტესტირება:
 lr = 0.01 -> Train Acc: 42% | Val Acc: 40%
 lr = 0.001 -> Train Acc: 65% | Val Acc: 49%
 lr = 0.0001 -> Train Acc: 44% | Val Acc: 44%

კრიტიკული ანალიზი: მოდელი განიცდის Underfitting-ს. პარამეტრების მცირე რაოდენობის (~149K) გამო ქსელს არ გააჩნია საკმარისი მოცულობა (capacity), რათა აღიქვას სახის გამომეტყველების რთული არაწრფივი კანონზომიერებები.

---

### 2. MediumCNN — გაფართოებული მოდელი
არქიტექტურა: ფენების რაოდენობა გაიზარდა 4-მდე (32 -> 64 -> 128 -> 128 ფილტრი). დაემატა Dropout (0.3 / 0.5) ტრენინგის დასასტაბილურებლად. FC ფენა: 1152 -> 256 -> 7.

#### ჰიპერპარამეტრების ტესტირება:
lr = 0.001, Dropout = 0.3 -> Train Acc: 69% | Val Acc: 54%
lr = 0.001, Dropout = 0.5 -> Train Acc: 64% | Val Acc: 55%

კრიტიკული ანალიზი: მოდელმა აჩვენა მკვეთრი Overfitting. მიუხედავად იმისა, რომ ტრენინგ სიზუსტე გარკვეულ ექსპერიმენტებში 90%-ს უახლოვდებოდა, ვალიდაციის სიზუსტე 55%-ზე გაიყინა. მოდელმა დაიწყო სავარჯიშო მონაცემების "ზეპირად დამახსოვრება" (memorization). Dropout(0.5)-ის შემოტანამ ნაწილობრივ შეამცირა ეს ეფექტი, მაგრამ პრობლემა მაინც თვალსაჩინო დარჩა.

---

### 3. DeepCNN + BatchNorm — დაბალანსებული მოდელი
 არქიტექტურა: 6 კონვოლუციური ფენა, სადაც თითოეულს მოსდევს Batch Normalization (BN). დაემატა Dropout2d კონვოლუციების შემდეგ და Dropout სრულად დაკავშირებულ (FC) ფენებში.
 მონაცემთა აუგმენტაცია (Data Augmentation): გამოიყენდა RandomHorizontalFlip, RandomRotation(10) და RandomAffine მოდელის გენერალიზაციის უნარის გასაზრდელად.
 ოპტიმიზაცია: Adam ოპტიმიზატორი წონების შემცირებით (weight_decay = 1e-4) და StepLR ლერნინგ რეიტის სკედულერი (gamma = 0.5, ყოველ 5 ეპოქაში).

#### ჰიპერპარამეტრების ტესტირება:
 lr = 0.001, Dropout = 0.5 -> Train Acc: 51% | Val Acc: 55%
 lr = 0.001, Dropout = 0.3 -> Train Acc: 53% | Val Acc: 58%

კრიტიკული ანალიზი: ქსელი გახდა დაბალანსებული. სავარჯიშო და ვალიდაციის სიზუსტეებს შორის სხვაობა მინიმალურია (53% vs 58%), რაც მიუთითებს იმაზე, რომ აუგმენტაციამ და BatchNorm-მა ეფექტურად აღმოფხვრეს ოვერფიტინგი. ბატჩ-ნორმალიზაციამ ქსელს მისცა საშუალება, უფრო მაღალი lr-ის პირობებშიც შეენარჩუნებინა გრადიენტების სტაბილურობა.

---

### 4. ResNetFER — საუკეთესო შედეგი (State-of-the-Art ამ პროექტში)
არქიტექტურა: სპეციალურად მორგებული რეზიდუალური ქსელი. საწყისი ფენა: Conv(1 -> 32) + BN + ReLU. მას მოსდევს 4 რეზიდუალური ბლოკი სკიპ-კავშირებით (Skip Connections): 32 -> 64 -> 128 -> 256 -> 256. ბოლოში გამოყენებულია AdaptiveAvgPool და FC ფენა: 256 -> 128 -> 7.
რეგულარიზაციის მოწინავე მეთოდები: ლოს-ფუნქციად გამოყენებულ იქნა CrossEntropyLoss ლეიბლების დაგლუვებით (Label Smoothing = 0.1). სასწავლო ტემპის დინამიკისთვის შეირჩა CosineAnnealingLR (T_max=20, eta_min=1e-5).

#### ჰიპერპარამეტრების ტესტირება:
* lr = 0.001, Dropout = 0.4 -> Train Acc: 67% | Val Acc: 65%
* lr = 0.001, Dropout = 0.2 -> Train Acc: 68% | Val Acc: 65%

> კრიტიკული ანალიზი: საუკეთესო მოდელი. სკიპ-კავშირებმა უზრუნველყვეს გრადიენტების იდეალური დინამიკა, ხოლო CosineAnnealingLR-მა იპოვა ლოკალური მინიმუმის ოპტიმალური წერტილი. მოდელი აჩვენებს საუკეთესო გენერალიზაციას ტესტ სეტზე მინიმალური სხვაობით ტრეინინგსა და ვალიდაციას შორის.

## ძირითადი მიგნებები (Key Findings)

1. Underfitting (SmallCNN): ტევადობის სიმცირის გამო მოდელი ვერ სწავლობს ფიჩერებს. გამოსავალი პარამეტრების გაზრდაა.
2. Overfitting (MediumCNN): პარამეტრების გაზრდა რეგულარიზაციის გარეშე იწვევს ხმაურის დაზეპირებას.
3. BatchNorm + Augmentation (DeepCNN): მონაცემთა ხელოვნური გამრავლება (Augmentation) არის მთავარი იარაღი ოვერფიტინგთან საბრძოლველად.
4. Skip Connections (ResNetFER): იძლევა უფრო ღრმა სტრუქტურების სტაბილურად გაწვრთნის საშუალებას და აუმჯობესებს სიზუსტეს 65%-მდე.
5. Label Smoothing და დისბალანსი: ლეიბლების დაგლუვებამ შეამცირა მოდელის ზედმეტი თავდაჯერებულობა (overconfidence) შეცდომების დაშვებისას, რაც განსაკუთრებით დაეხმარა მცირე კლასების (Disgust) პროგნოზირებას.
